2017.05.27

1.
虽然用了很久的 linux, 但我不得不承认我的技术还是很差的, 我连 shell 脚本都还不会写, makefile 也学得很泛, find, grep, vi, awk, xargs, mount 等常用命令也还不怎么熟悉, 对内核原理一知半解, 所以我打算先把 CommonLisp 的学习先放一放, 先打基础
2.
其余的时间学英语和数学, 一定一定

2017.05.26

1.
花了一上午把 sba(全称是:sb-daemon) 强化了, 除了整理代码外, 第一, 我加入了平滑调节, 现在不会突然升高和降低, 产生闪烁的效果了, 第二, 守护程序每半个小时运行一次, 等亮度稳定, 再陷入休眠, 避免长时间占用设备, 在使用上, 我建议也不要把摄像头对着窗外, 可以让他对着天花板, 或放进一个白色纸箱里, 如此画面复杂度较低, 可以准确地检测环境亮度等级

2017.05.25

1.
本来我写 C 程序做相等判断一直是(常量==变量)风格的, 这么做的好处是, 如果你把等号错误地写成了赋值, 编译器会报错, 可我最近作死换了回来, 今天 mmap 一直不成功, 头疼了好几个小时, 最后发现是把条件写成了(pointer = MAP_FAILED), 看来前辈们总结的经验不无道理啊
2.
看到 logitech 的 Webcam 吃灰那么久, 突发奇想, 不如把它当成 sensor 自动调节屏幕亮度吧, 但是效果并不好, 原因是摄像头会根据环境亮度自动调节 ISO, 特别是在晚上, 屏幕光打在我脸上, 会让屏幕的亮度发了疯一般升高
3.
问题1: 给 gcc 加 -Wall 参数, 问题2: 不要把摄像头对着脸, 对着窗外

2017.05.25

1.
很长时间没有登录微软的系统了, 用 linux 越多, 越能体会到命令行和脚本的先进性, 自动化工作带给人的愉悦和满足感是鼠标和控件永远比不上的, 也是只属于程序员的宝贝, 等我下次整顿电脑或者换电脑的时候, 那样的系统可能将永远停留在虚拟机中了:)

2017.05.24
